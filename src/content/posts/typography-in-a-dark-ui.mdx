深色博客最容易犯的错，是把所有东西都做得很亮、很满、很重。结果不是更“技术”，而是更难读。

## 标题和正文必须拆层

技术博客的阅读节奏通常很明确：先扫标题，再扫摘要，最后决定要不要进入正文。

如果标题、摘要、标签、日期都用接近的亮度和粗细，页面就会失去层级。首版里我把这几层拆成了：

- 标题用高对比
- 摘要降一级
- 元信息再降一级
- 标签保持可点击，但不抢正文

<InlineNote title="Design note">
  只要层级清楚，页面就可以很克制，不需要靠夸张动效去解释结构。
</InlineNote>

## 代码块要和正文区分，但不要刺眼

代码块是阅读中的停顿点。它需要被一眼识别出来，但不应该像广告位一样跳出来。

```ts
export function withBase(pathname: string): string {
  const path = pathname.replace(/^\/+|\/+$/g, "");
  return path ? `/blog/${path}/` : "/blog/";
}
```

这里更重要的是边框、背景和滚动行为是否稳定，而不是加多少炫技效果。

## 深色不等于纯黑

纯黑背景在长文阅读里会让亮色文字对比过头，眼睛很容易累。略微抬起底色，再给分层容器少量透明度，通常会更耐看。
